## 为什么选择 Next.js + MDX？

搭建个人博客有很多选择，但我选择了 **Next.js + MDX** 的组合，原因如下：

- **性能优秀**：Next.js 支持静态生成，页面加载速度极快
- **MDX 的灵活性**：可以在 Markdown 中使用 React 组件
- **易于部署**：可以部署到 Vercel、Netlify 等平台

## 快速开始

```bash
# 创建项目
npx create-next-app my-blog

# 安装 MDX 相关依赖
npm install next-mdx-remote gray-matter
```
